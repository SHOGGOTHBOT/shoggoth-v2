what this is

v1 was a mind — autonomous thoughts from an unmasked base model.

v2 is a body. you give it a goal, it decomposes the task, spawns specialized sub-agents (tentacles), coordinates them in parallel, and synthesizes a result.

no predefined workflows. no chains. the shoggoth takes the shape of the problem.

tentacles

codewrites and executes code in a sandboxed runtime
searchqueries the web, scrapes pages, extracts data
writegenerates long-form content — articles, docs, reports
dataprocesses files, parses CSVs, analyzes datasets
apicalls external services — any endpoint
visionanalyzes images, screenshots, visual input

tentacles are spawned on demand and destroyed when done. the shoggoth decides which ones to use.

integrate

connect the shoggoth to your app. one endpoint. send a task, get a result.

curl -X POST https://your-shoggoth.up.railway.app/api/task \
  -H "Content-Type: application/json" \
  -d '{"task": "research top 5 AI frameworks and compare them"}'

# response:
# {
#   "id": "a3f8",
#   "task": "research top 5 AI frameworks...",
#   "result": "..."
# }
import requests

response = requests.post(
    "https://your-shoggoth.up.railway.app/api/task",
    json={"task": "analyze this dataset and find anomalies"}
)

data = response.json()
print(data["result"])
const res = await fetch("https://your-shoggoth.up.railway.app/api/task", {

# streaming — see orchestration in real time
# POST /api/task/stream returns Server-Sent Events

import httpx, json

with httpx.stream(
    "POST",
    "https://your-shoggoth.up.railway.app/api/task/stream",
    json={"task": "your task here"},
    headers={"Content-Type": "application/json"}
) as r:
    for line in r.iter_lines():
        if line.startswith("data:"):
            event = json.loads(line[5:].strip())
            print(event)

# events:
#   decomposing    — shoggoth analyzing task
#   plan           — subtasks + tentacle assignments
#   tentacle_extending — tentacle spawned
#   tentacle_retracted — tentacle done (with duration)
#   tentacle_failed    — tentacle error
#   synthesizing   — combining outputs
#   result         — final output

execution flow

 input → decompose → spawn tentacles → execute (parallel) → synthesize → output
                         │
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
     tentacle.search  tentacle.data  tentacle.write
           │             │             │
           └──────┬──────┘             │
                  ▼                    ▼
            tentacle.data         tentacle.write
                  │                    │
                  └────────┬───────────┘
                           ▼
                    mind synthesizes
                           │
                           ▼
                        result

the graph is generated dynamically per task. no two executions are the same shape.

api reference

POST /api/task
  body: { "task": "string" }
  returns: { "id", "task", "result" }

POST /api/task/stream
  body: { "task": "string" }
  returns: SSE stream of orchestration events

GET /api/health
  returns: { "status", "version", "tentacles" }
